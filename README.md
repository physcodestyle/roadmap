# Дорожная карта развития компетенций Flutter-разработчика

Эта дорожная карта содержит список знаний, умений и навыков, необходимых Flutter-разработчику на разных этапах карьеры. Кривая обучения разбита по уровням и создана на основе опыта работы участников проекта. Основой для дорожной карты служит список компетенций, которые также привязаны к уровням. Дорожная карта не только служит ориентиром на пути развития Flutter-разработчика, но и содержит список актуальных источников информации, учебные материалы, проверочные тесты и задокументированный опыт участников проекта.


> Список требований к определенным уровням разработчика сильно зависят от компании. Например, для одной компании стажеры — это новички, которые знают язык на каком-то уровне и умеют решать алгоритмические задачи, для другой, стажеры — это разработчики на испытательном сроке. Все описанные требования являются усредненными по рынку. При подготовке к собеседованию необходимо отталкиваться от требований конкретной компании и не стоит останавливаться на достигнутом.


## Требования к уровню «Стажер (Trainee)»

От стажера в первую очередь требуется хороший уровень знаний в области компьютерных наук, хороший уровень владения языком Dart, понимание его особенностей, понимаение основных возможностей фреймворка Flutter. Плюсом является наличие собственных проектов, желательно каких-то нестандартных — лучше проявить творческие способности, насмотренность, оригинальность.

### Группа компетенций по компьютерным наукам

> Для уровней «Стажер» и «Младший разработчик» данные компетенции практически идентичны. В разных компаниях требования к глубине знаний, умений и навыков разняться. Например, в каких-то компаниях об алгоритмах вообще практически не спрашивают, в других — вводят специальные алгоритмические секции или даже отдельные собеседования.

1. **Алгоритмы и структуры данных:**
  - знать определения следующих cтруктур данных:
    - массив,
    - связный список,
    - стек,
    - очередь,
    - граф,
    - дерево;
  - уметь реализовать следующие алгоритмы:
    - линейный поиск (массив),
    - бинарный поиск (массив),
    - сортировка выбором (массив),
    - сортировка пузырьком (массив),
    - быстрая сортировка (массив),
    - поиск в ширину (дерево),
    - поиск в глубину (дерево),
    - рекурсия;
  - знать особенности использования описанных выше алгоритмов и структур данных.
2. **Объектно-ориентированное программирование:**
  - знать определения основных принципов ООП — наследование, инкапсуляция, полиморфизм, абстракция;
  - уметь использовать классы для построения и работы с предметной областью;
  - знать особенности работы с ООП.
3. **Чистая архитектура:**
  - знать принципы SOLID,
  - уметь применять принципы SOLID для рефакторинга,
  - знать особенности использования SOLID в языке Dart.

### Группа компетенций по работе с проектами

1. **Система контроля версий Git:**
  - знать основные понятия, которые используются в системах контроля версий:
    - определение, цель и задачи системы контроля версий,
    - классификация систем контроля версий и место Git в ней,
    - ветки и особенности их использования;
  - уметь использовать Git:
    - создавать репозитории,
    - отслеживать изменения,
    - сохранять изменения,
    - отправлять изменения в удаленный репозиторий,
    - откатывать изменения,
    - создавать ветки,
    - объединять ветки.

### Группа компетенций по работе с Dart

> Для уровней «Стажер» и «Младший разработчик» данные компетенции практически идентичны, требуются очень хорошие знания и навыки использования языка. Эти компетенции получить сложнее, чем компетенции по работе с Flutter.

1. **Типы данных и переменные:**
  - знать все типы данных и их отличия;
  - знать все виды переменных и констант и их отличия (`final`, `const`, `var`, `late`);
  - знать для чего используется ключевое слово `dynamic`;
  - знать о механизме `null-safety`, дереве типов (операторы `??`, `.?`, `.!`, `…?`).
2. **Структурное программирование:**
  - знать ключевые слова и конструкции:
    - операторы присваивания, сравнения, сдвига, арифметических действий;
    - конструкции для организации ветвления (`if-else`, `switch..case`, `?:`);
    - все виды циклов (`while`, `do..while`, `for`, `for-each`, `for-in`);
    - методы анализа и обработки структур данных (`pattern matching`);
  - уметь работать с функциями;
  - владеть навыками структурного программирования в Dart.
3. **Работа с коллекциями:**
  - знать и уметь использовать типы коллекций (`List`, `Map`, `Set`);
  - знать и уметь использовать (опционально) типы коллекций из библиотеки `dart:collections`.
4. **Объектно-ориентированное программирование:**
  - знать все типы конструкторов;
  - знать как реализуется полиморфизм разных типов;
  - знать как реализуется наследование;
  - уметь учитывать особенности реализации классов;
  - уметь использовать ключевое слово `this`, , `mixins`, `extensions` и модификаторы классов.
6. **Асинхронность и потоки:**
  - знать как используются конструкции `async-await`, `Future`, `Stream`.
7. **Обработка ошибок:**
  - знать как устроена обработка ошибок в коллбэках и в блоке кода (`try-catch-finally`);
  - знать отличия типов `Error` и `Exception`.
8. **Работа с сетью:**
  - знать возможности библиотеки `dart:io` (`HttpServer`, `HttpRequest`, `HttpResponse`);
  - уметь получать данные из сети по протоколам `HTTP` и `HTTPS`.

### Группа компетенций по работе с Flutter

> Очень часто для уровня «Стажер» требуют немного в плане знания фреймворка, достаточно знать базовые вещи и уметь верстать простые интерфейсы.

1. **Виджеты:**
  - знать отличия `Stateful`- и `Stateless`-виджетов;
  - уметь изменять состояние виджетов;
  - уметь использовать стандартные виджеты для верстки простых интерфейсов:
    - `Scaffold`,
    - `MaterialApp`,
    - `SizedBox`,
    - `Container`,
    - `Column`,
    - `Row`,
    - `ListView`,
    - `GridView`,
    - `FutureBuilder`;
  - уметь использовать функцию `runApp`;
2. **Навигация пользователя в приложении:**
  - знать типы навигации (безымянная, навигация по имени, генерированная навигация);
  - уметь осуществлять навигацию пользователя и передавать параметры между экранами.

### Группа компетенций по работе с Flutter

> В отличие от предыдущих 2-х частей, пункт Flutter для Junior разработчика сильно отличается от соответвующего пункта для Trainee. В первую очередь тем, что здесь уже требуется наличие и опыт написания пет-проектов, более глубокое понимание фреймворка и опыт работы с Менеджерами Состояний.

0. Все знания, умения и навыки предыдущего уровня.
1. **Виджеты:**
  - знать об устройстве деревьев во Flutter;
  - знать об особенностях отрисовки и управлении жизненным циклом виджетов ([Flutter под капотом](https://habr.com/ru/articles/501862/));
  - уметь использовать методы жизненного цикла виджетов `Stateful`-виджета.
  - знать как использовать наследуемые виджеты (Inherited Widgets);
  - уметь пользоваться библиотекой `Provider`;
  - уметь использовать продвинутые виджеты для верстки сложных интерфейсов.
2. **Навигация пользователя в приложении:**
    - знать об особенностях декларативного навигатора;
    - уметь работать с Router, его компонентами а также готовыми библиотеками ([auto_route](https://pub.dev/packages/auto_route), [go_router](https://pub.dev/packages/go_router), [Octopus](https://pub.dev/packages/octopus)).
3. **Интерфейсы:**
    - знать основные правила лэйаута во Flutter
    - знать концепции ограничений и размеров экранов.
4. **Использование билдеров (Builders):**
    - `FutureBuilder`, `StreamBuilder`, `ListnableBuilder`. Уметь реактивно управлять интерфейсами.
5. **Работа с большими списками:**
  - уметь использовать `ListView.builder` и `cacheExtent` для ленивой подгрузки списков большого или неограниченного размера;
  - уметь реализовать постраничного вывода списков (пагинацию).
6. **Ключи (Keys):**
  - знать все виды ключей и для чего они используются.
7. **Управление состоянием (State Management):**
  - знать как минимум одну из наиболее популярных библиотек для управления состоянием:
    - [BLoC](https://pub.dev/packages/flutter_bloc) (Business Logic Component, Компонент Бизнес Логики) — золотой стандарт во Flutter для управления состоянием (большинство проектов используют именно эту библиотеку),
    - [Redux](https://pub.dev/packages/redux) — популярный менеджер, перетекший к нам из веба, построенный на архитектурной концепции MVI и иммутабельной модели данных,
    - [MobX](https://pub.dev/packages/mobx) — популярный менеджер, построеный на мутабельной модели,
    - [Elementary](https://pub.dev/packages/elementary) — архитектурный пакет, построенный на коцепции `MVVM` (`MWWM` во Flutter) для отделения бизнес логики от View,
    - [Riverpod](https://pub.dev/packages/riverpod) — популярное решение для управления состоянием во Flutter, kогичное продолжение идей Provider;
  - (опционально) уметь управлять состоянием приложения без использования сторонних библиотек.
